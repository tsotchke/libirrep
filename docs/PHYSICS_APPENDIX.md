# Physics conventions — authoritative appendix

This document fixes the mathematical conventions to which every result
computed by libirrep conforms. Each section names the convention, states
its formal definition, derives the consequential identities the library
relies on, and cites the primary sources. A user whose own code deviates
from any convention here — even by a sign — will obtain results that
superficially pass sanity checks but are systematically wrong; the only
remedy is to match exactly or to convert explicitly at the interface.

Symbols used throughout:

| Symbol | Meaning |
| ------------- | ----------------------------------------------------------- |
| `ℝ`, `ℂ` | the real, complex fields |
| `SO(3)` | special orthogonal group in 3D (proper rotations) |
| `SU(2)` | 2×2 complex unitary matrices with determinant 1 |
| `O(3)` | full orthogonal group in 3D (rotations + reflections) |
| `j`, `l` | angular-momentum / spin-multiplet label (`2j ∈ ℤ`, `l ∈ ℤ≥0`) |
| `m` | magnetic quantum number, `m ∈ {−j, −j+1, …, +j}` |
| `p` | parity index, `p ∈ {+1, −1}` |
| `D^j(R)` | Wigner-D matrix of R in the spin-j irrep |
| `d^j(β)` | small Wigner-d matrix at Euler middle angle β |
| `Y_l^m` | complex spherical harmonic |
| `Y_{l,m}^real`| real spherical harmonic |
| `⟨… | …⟩` | Clebsch-Gordan coefficient |

All citations are collected in [`docs/REFERENCES.md`](REFERENCES.md); bare
author-year references here are shorthand for the entries there.

## 1. Rotation groups

### 1.1. SO(3)

`SO(3)` is the Lie group of orientation-preserving orthogonal transformations
of ℝ³ — equivalently, 3×3 real matrices `R` satisfying `Rᵀ R = I` and
`det R = +1`. As a manifold it is the real projective 3-space ℝP³ (Hall
2015, §1.2), compact and connected with fundamental group `π₁(SO(3)) = ℤ₂`.
The library represents elements of SO(3) in four equivalent forms — rotation
matrix, unit quaternion, Euler ZYZ triple, axis-angle — with exact
conversions tested to `1e-12` relative error (round-trip; see
`tests/test_so3.c`).

### 1.2. SU(2) and the double cover

`SU(2)` is the group of 2×2 complex unitary matrices `U = ᵢ exp(−iθ n̂·σ/2)`
with `det U = 1`, where `σ = (σ_x, σ_y, σ_z)` are the Pauli matrices. As a
manifold it is the 3-sphere `S³`, simply connected. There is a surjective
two-to-one homomorphism
```
 π : SU(2) → SO(3), U ↦ R(U),
```
where `R(U)_{ij} = ½ Tr(σ_i U σ_j U†)` (Sakurai & Napolitano 2020, §3.3;
Hall 2015, §1.3). Both `U` and `−U` project to the same `R`, so SU(2) is
the **universal double cover** of SO(3), and `SU(2) / ℤ₂ ≅ SO(3)`.

This double cover is why spin-½ particles acquire a sign under rotation by
`2π` (Sakurai §3.2): the SU(2) element `U(2π n̂) = −I` projects to
`R(2π n̂) = I`, so the classical rotation is trivial but the quantum state
acquires a phase of `−1`. The library explicitly exercises this invariant
in `examples/spin_half_rotation.c`.

### 1.3. O(3) and parity

`O(3)` is the full orthogonal group including reflections, the direct product
`O(3) ≅ SO(3) × ℤ₂` where the ℤ₂ factor is generated by inversion
`P : r ↦ −r`. Irreducible representations of O(3) are indexed by `(l, p)`
with `l ∈ ℤ≥0` and `p ∈ {+1, −1}`; a rotation `R ∈ SO(3)` acts on `(l, p)`
as `D^l(R)`, while inversion acts as `p · I`. The library uses `p = +1` for
"even" / "e" irreps (polar scalars, axial vectors) and `p = −1` for "odd" /
"o" (polar vectors, axial scalars / pseudoscalars); this is the e3nn
convention (Geiger et al. 2022) and matches the multiset parser accepted
by `irrep_multiset_parse`.

## 2. Rotation parameterisations

### 2.1. Euler angles (ZYZ)

The ZYZ Euler decomposition writes every `R ∈ SO(3)` as the active product
```
 R(α, β, γ) = R_z(α) · R_y(β) · R_z(γ), α ∈ [0, 2π), β ∈ [0, π], γ ∈ [0, 2π) (2.1)
```
reading right to left (the inner rotation `R_z(γ)` acts first on a vector
being rotated). This is the physics / quantum-mechanics convention adopted
by Sakurai §3.3, Varshalovich §1.4, and Edmonds §1.2. The three factors
satisfy
```
 R_z(θ) = [[cos θ, −sin θ, 0], [sin θ, cos θ, 0], [0, 0, 1]],
 R_y(θ) = [[cos θ, 0, sin θ], [0, 1, 0], [−sin θ, 0, cos θ]].
```
The parameterisation is two-to-one at the boundaries `β = 0` and `β = π`,
known as **gimbal lock**: at `β = 0` only `α + γ` is determined, at `β = π`
only `α − γ` is. The inverse map `irrep_euler_zyz_from_rot` collapses
`γ → 0` in those regions and applies `|sin β| < 10⁻¹²` as the numerical
threshold, with an explicit justification in
`include/irrep/so3.h` (`IRREP_EULER_GIMBAL_EPS`). Users whose pipeline
depends on distinguishing `α` and `γ` near the poles must avoid ZYZ and
use quaternion or axis-angle.

### 2.2. Quaternions

A unit quaternion `q = (x, y, z, w)` with `x² + y² + z² + w² = 1` represents
the rotation
```
 q = sin(θ/2) · n̂ + cos(θ/2) · ê₀, (2.2)
```
by angle `θ` about unit axis `n̂`, where `ê₀` is the identity (scalar
one). The mapping `q → R(q)` is explicitly
```
 R(q) = [[1−2(y²+z²), 2(xy−zw), 2(xz+yw) ],
 [ 2(xy+zw), 1−2(x²+z²), 2(yz−xw) ],
 [ 2(xz−yw), 2(yz+xw), 1−2(x²+y²)]] (2.3)
```
(Shuster 1993, §II.A). The library stores quaternions with scalar-last
layout `{x, y, z, w}`. This is a deliberate choice for SIMD ergonomics —
the vector part occupies the first three lanes of a `float32x4_t` /
`float64x4_t` — and matches the Eigen, glTF 2.0, and Unreal Engine
conventions. Some astronomy and quantum-chemistry code uses scalar-first
`{w, x, y, z}`; consumers interoperating with such code must permute
explicitly at the boundary.

The unit sphere `S³` in ℝ⁴ is the underlying manifold of SU(2), and
`q ↦ R(q)` is the explicit form of `π : SU(2) → SO(3)`. Both `q` and `−q`
produce identical rotation matrices. Shoemake's uniform sampler
(`irrep_quat_random`; Shoemake 1992) canonicalises the returned sample to
have `w ≥ 0` so that consumers doing sign-dependent arithmetic get a
consistent branch.

### 2.3. Axis-angle

The axis-angle representation uses a unit vector `n̂ ∈ S²` and an angle
`θ ∈ [0, π]`. Rodrigues' rotation formula
```
 R(n̂, θ) = I + sin(θ) · [n̂]_× + (1 − cos(θ)) · [n̂]_×², (2.4)
```
where `[n̂]_× = [[0, −n_z, n_y], [n_z, 0, −n_x], [−n_y, n_x, 0]]` is the
matrix of the cross product with `n̂`, gives the explicit conversion. The
library's `irrep_rot_exp(ω)` implements Rodrigues with a first-order
Taylor fallback for `|ω| < 10⁻¹²`, since the full formula has `sin(θ)/θ`
and `(1 − cos θ)/θ²` factors that cancel catastrophically in that regime.

### 2.4. Matrix exponential and logarithm on SO(3)

The exponential map `exp : so(3) → SO(3)` sends a skew-symmetric
generator `ω̂ ∈ so(3)` to `exp([ω̂]_×) = R(ω̂, |ω|)`, i.e. Rodrigues'
formula. The logarithm is its local inverse on a neighbourhood of the
identity, but `SO(3)` is not simply connected: the closed-form
extraction
```
 ω = (θ / (2 sin θ)) · axial(R − Rᵀ), θ = arccos((Tr R − 1)/2) (2.5)
```
has a removable singularity at `θ = 0` handled by l'Hôpital and a
genuine branch ambiguity at `θ = π`, where `R = Rᵀ` forces
`axial(R − Rᵀ) = 0` and the axis must instead be recovered from the
symmetric part `R + Rᵀ − (Tr R − 1) I = 2 n̂ n̂ᵀ`. Libirrep's
`irrep_rot_log` does **not** evaluate (2.5) directly; it extracts a
unit quaternion via Shepperd's four-branch algorithm (Shepperd 1978)
and applies Markley's branch-switching axis-angle recovery
(Markley 2008, §III). This produces the correct axis for every input
including the `θ = π` locus, with accuracy verified to `1e-12`
relative error across the unit sphere of inputs
(`tests/test_so3.c`). Equation (2.5) appears here for pedagogical
context and is **not** the implemented algorithm.

### 2.5. Quaternion-from-matrix (Shepperd)

The map `R → q` is well-defined up to sign on SO(3). Naïve extraction via
`w = ½ √(1 + Tr R)` fails when `Tr R → −1`. Shepperd's method (Shepperd
1978) switches among four candidate recoveries based on the largest of
`Tr R`, `R_{00}`, `R_{11}`, `R_{22}`, guaranteeing stability for every
`R ∈ SO(3)`. Libirrep's `irrep_quat_from_rot` implements Shepperd's
algorithm verbatim.

## 3. Irreducible representations of SO(3) and SU(2)

### 3.1. Spin-j irreps

For each `2j ∈ ℤ≥0` there is an irreducible representation of SU(2) of
complex dimension `2j + 1`. Basis states `|j, m⟩` are eigenstates of
`J_z` with `J_z |j, m⟩ = m |j, m⟩`, `J² |j, m⟩ = j(j+1) |j, m⟩`, and
`m ∈ {−j, −j+1, …, +j}`. Integer-j irreps are also irreps of SO(3);
half-integer irreps are projective representations of SO(3) (the sign
flip under a `2π` rotation cannot be absorbed), well-defined as SU(2)
irreps only.

Libirrep supports `j ≤ 16` (i.e. `IRREP_TWO_J_MAX = 32`), with all
half-integer APIs disambiguated by the `_2j` suffix (`irrep_cg_2j`,
`irrep_wigner_d_small_2j`, `irrep_wigner_6j_2j`, …) taking **doubled**
integer arguments so that spin-½ enters as `two_j = 1`.

### 3.2. The `(l, p)` label for O(3)

For the orthogonal group, the irrep `(l, p)` acts on the same `(2l+1)`-
dimensional space as the SO(3) irrep of label `l`, with inversion
multiplying by `p ∈ {±1}`. The two conventions commonly seen are:
- **e3nn / libirrep**: `+1` = even (`e`), `−1` = odd (`o`). Polar vectors
 (e.g. position) have `p = −1`; axial vectors (e.g. angular momentum,
 magnetic moment) have `p = +1`.
- **Spectroscopic** (`g`, `u`): `g` = gerade (even), `u` = ungerade (odd).
 Equivalent, just different letters.

Libirrep accepts `"1x1o"` for one copy of the polar-vector irrep.

## 4. Phase and sign conventions

### 4.1. Condon–Shortley phase

The associated Legendre polynomial `P_l^m(x)` with Condon–Shortley phase
is
```
 P_l^m(x) = (−1)^m · (1 − x²)^{m/2} · (dᵐ / dxᵐ) P_l(x), (4.1)
```
where `P_l(x)` is the unsigned Legendre polynomial (Sakurai Appendix A;
Jackson 3rd ed., §3.6). The complex spherical harmonic follows as
```
 Y_l^m(θ, φ) = √{(2l+1)/(4π) · (l−m)!/(l+m)!} · P_l^m(cos θ) · e^{imφ}. (4.2)
```
Libirrep applies the `(−1)^m` factor exactly once, in `irrep_legendre_assoc`;
the Wigner-D construction does **not** reintroduce it. Applying the phase
in the angular exponential instead would double-count it — the two
conventions differ by a `(−1)^m` factor — so the placement is a
convention choice, not an algebraic identity. With the phase localised
to `P_l^m`, the conjugation relation `Y_l^{−m} = (−1)^m (Y_l^m)*`
follows with no additional phase accounting, matching Sakurai
(Appendix A) and Jackson (§3.6).

### 4.2. Real spherical harmonics (e3nn convention)

Real-basis SH are built from the complex ones via the unitary transform
```
 Y_{l, 0}^{real} = Y_l^0,
 Y_{l, +|m|}^{real} = √2 · ℜ(Y_l^{|m|}) = (Y_l^{|m|} + Y_l^{−|m|}) · 1/√2 · (phase),
 Y_{l, −|m|}^{real} = √2 · ℑ(Y_l^{|m|}) = (Y_l^{|m|} − Y_l^{−|m|}) · i/√2 · (phase). (4.3)
```
The phase is fixed so that `Y_{1, +1}^{real} ∝ +x` at the equator (Geiger
et al. 2022, Appendix A); this is the convention shared by the e3nn,
Wikipedia "Real spherical harmonics" article, and libirrep. Some
quantum-chemistry codes (ORCA, NWChem) use a different sign on the odd-m
real components; the change-of-basis is a diagonal `±1` matrix that the
consumer must apply explicitly.

### 4.3. Quaternion layout

Covered in §2.2. Libirrep's `{x, y, z, w}` layout with `w` last is the
SIMD-ergonomic convention of Eigen, glTF, Unreal Engine; the
`{w, x, y, z}` convention (scalar-first) is seen in astronomy and some
graphics libraries. Document interop at the boundary.

### 4.4. Active versus passive rotations

An **active** rotation `R` sends a vector `v` to `R v`, leaving the
coordinate axes fixed. A **passive** rotation rotates the axes by `−θ`
(equivalently, applies `R⁻¹` to the vector expressed in the new axes).
Libirrep uses active rotations throughout: when `R = R_z(θ)`, the vector
`x̂` becomes `(cos θ, sin θ, 0)`, not `(cos θ, −sin θ, 0)`. Passive
conventions appear in older crystallography and aerospace work; translate
by inverting.

## 5. Spherical harmonics

The orthonormality relation
```
 ∫_{S²} Y_l^m(r̂) · [Y_{l'}^{m'}(r̂)]* dΩ = δ_{l, l'} δ_{m, m'} (5.1)
```
holds on the unit 2-sphere with the standard measure `dΩ = sin θ dθ dφ`.
Libirrep verifies orthonormality to `1e-10` via Lebedev–Laikov quadrature
of order 35 (`tests/test_spherical_harmonics.c`; rule tabulated in
`src/quadrature_lebedev_data.c` with node and weight values from Lebedev
& Laikov 1999, public domain).

The **addition theorem**
```
 P_l(cos γ) = (4π / (2l + 1)) · Σ_m Y_l^m(r̂₁) · [Y_l^m(r̂₂)]* (5.2)
```
(Jackson §3.6, Arfken 7th ed. §15.6) links SH evaluated at two unit
vectors to the Legendre polynomial of their dot product. Libirrep's
addition-theorem test locks the absolute residual below `5 × 10⁻¹⁴` for
`l ≤ 8`, which is the minimum precision at which the real-basis
normalisation is verified empirically.

### 5.1. Associated Legendre via stable recurrence

The direct formula (4.1) has a derivative that is catastrophically
ill-conditioned at `|x| → 1` for large `l`. Libirrep instead applies the
three-term recurrence
```
 P_l^m(x) = [ (2l − 1) x · P_{l−1}^m(x) − (l + m − 1) · P_{l−2}^m(x) ] / (l − m) (5.3)
```
(Press et al. 2007, §6.7; Limpanuparb & Milthorpe 2014, Eq. 6), evaluated
forward from the closed-form initialisers
```
 P_m^m(x) = (−1)^m · (2m − 1)!! · (1 − x²)^{m/2}, (5.4)
 P_{m+1}^m(x) = x · (2m + 1) · P_m^m(x). (5.5)
```
The forward recurrence in `l` is numerically stable for `x ∈ [−1, 1]`
(Limpanuparb & Milthorpe §2), preserving full double precision up to
`l = 50` in our testing. Implementation-level performance optimisations
(including the triangular-grid amortisation of the recurrence in the
NEON batch kernel) are discussed in [`DESIGN.md`](DESIGN.md) §2; for
bit-exactness against the scalar path, `#pragma STDC FP_CONTRACT OFF`
is applied in both the scalar and NEON translation units so fused
multiply-add selection does not produce a one-ulp divergence between
the two.

## 6. Wigner D-matrices

### 6.1. Factorisation via Euler angles

For `R = R_z(α) R_y(β) R_z(γ) ∈ SO(3)` and spin `j`, the Wigner-D matrix
factors as
```
 D^j_{m', m}(α, β, γ) = e^{−i m' α} · d^j_{m', m}(β) · e^{−i m γ}, (6.1)
```
where `d^j(β)` is the **small Wigner-d matrix**, real and depending only
on the middle Euler angle. The exponentials introduce the α, γ phase
dependence and are straightforward; the numerical content is concentrated
in `d^j`.

### 6.2. Small-d: Jacobi-polynomial form

For `m ≥ |m'|`, Edmonds (4.1.23) gives
```
 d^j_{m', m}(β) = √{(j+m)!(j−m)! / (j+m')!(j−m')!}
                · (cos β/2)^{m+m'} · (sin β/2)^{m−m'}
                · P_{j−m}^{(m−m', m+m')}(cos β).            (6.2)
```
Other `(m, m')` quadrants reduce to the canonical one via the
Varshalovich §4.4.1 symmetries — swap `m ↔ m'` with a `(−1)^{m−m'}`
phase, and/or take `(m, m') → (−m', −m)`.

The Jacobi polynomial `P_n^{(α,β)}(x)` itself is evaluated by the
NIST DLMF §18.9.1 forward three-term recurrence in `n`, which is
stable for non-negative integer `(α, β)` at `x ∈ [−1, 1]`. The sqrt
factorial ratio is computed in log-space via
`exp(½·(lgamma − lgamma − lgamma + lgamma))`, bounded in `j` only by
the IEEE-754 `lgamma` overflow limit (`j ≈ 170`).

The earlier direct-sum implementation (Sakurai 3.8.33) was replaced
here because the alternating-sign summation loses precision to
catastrophic cancellation past `j ≈ 20`: measured unitarity at
`(α,β,γ) = (0.3, 0.9, 1.5)` was `2 × 10⁻³` at `j = 50` and divergent
past `j = 60`. The Jacobi form measures `≤ 1 × 10⁻¹²` for every
`j ≤ 80` we have tested, limited only by the `O((2j+1)²)`
accumulation in the unitarity product itself.

Closed-form small-d values at `j ∈ {½, 1, 3/2, 2}` (Varshalovich
§4.16) agree to machine precision in `tests/test_wigner_d.c`.

### 6.3. Unitarity and composition

`D^j(R)` is unitary: `D^j(R) · D^j(R)† = I`. Composition satisfies
`D^j(R_1 R_2) = D^j(R_1) · D^j(R_2)`. Both identities are tested against
random SO(3) pairs for `j ∈ {1, 2, 3, 4, 6}` to `1e-8` absolute residual
in `tests/test_wigner_d.c`.

### 6.4. Derivative `∂d/∂β`

Needed for equivariant force evaluation in spin-based LLG integrators
(Batzner et al. 2022, §Methods-Forces). The derivative is obtained by
differentiating (6.2) term-by-term; the `k`-sum structure is preserved,
and per-term finite differences confirm agreement to `1e-8` across
`j ∈ {1, 2, 3, 4}` (`tests/test_wigner_d.c`).

## 7. Clebsch-Gordan coefficients

### 7.1. The Racah single-sum formula

`⟨j_1 m_1; j_2 m_2 | J M⟩` is the unitary change-of-basis coefficient from
the product space `|j_1, m_1⟩ ⊗ |j_2, m_2⟩` to the coupled space
`|(j_1 j_2) J, M⟩`. Racah (1942) established the closed single-sum form
```
 ⟨j_1 m_1; j_2 m_2 | J M⟩ =
 δ_{M, m_1 + m_2} · Δ(j_1, j_2, J) · √{(2J+1)} · √{N(j_1, m_1; j_2, m_2; J, M)}
 · Σ_k (−1)^k / [ k! (j_1 + j_2 − J − k)! (j_1 − m_1 − k)! (j_2 + m_2 − k)! (J − j_2 + m_1 + k)! (J − j_1 − m_2 + k)! ], (7.1)
```
where `Δ(j_1, j_2, J) = √{(j_1+j_2−J)! (j_1−j_2+J)! (−j_1+j_2+J)! / (j_1+j_2+J+1)!}`
is the triangle coefficient, `N(·)` is the product of the four `(j ± m)!`
factorials, and the summation range of `k` is the intersection of six
non-negativity conditions. The formula vanishes automatically when any
selection rule is violated. Libirrep implements (7.1) in log-gamma form
(`src/clebsch_gordan.c`) for numerical stability past `j ≤ 50` in double
precision.

### 7.2. Selection rules

`⟨j_1 m_1; j_2 m_2 | J M⟩ = 0` unless all four conditions hold
(Varshalovich §8.1):
```
 |j_1 − j_2| ≤ J ≤ j_1 + j_2, (triangle)
 m_1 + m_2 = M, (z-component)
 |m_i| ≤ j_i, |M| ≤ J, (ranges)
 2(j_1 + j_2 + J) ∈ 2ℤ (integer-sum of doubled labels).
```
Libirrep returns exactly `0.0` in every violation case, so callers may
sum freely over index ranges without explicit rule checks.

### 7.3. Relation to the Wigner 3j symbol

The Wigner 3j symbol has cleaner column symmetry (invariant up to a sign
under permutation of columns). The conversion is
```
 (j_1 j_2 j_3 ; m_1 m_2 m_3) =
 (−1)^{j_1 − j_2 − m_3} / √{2j_3 + 1} · ⟨j_1 m_1; j_2 m_2 | j_3 (−m_3)⟩. (7.2)
```
Libirrep provides both via `irrep_cg_2j` and `irrep_wigner_3j_2j`; the
cross-check `tests/test_clebsch_gordan.c` validates (7.2) for
`(j_1, j_2) ∈ {(½, ½), (1, ½), (1, 1), (3/2, 1), (2, 2), (3, 2)}`
against hand-tabulated values (Sakurai Appendix A) to machine precision.

### 7.4. Half-integer support

Every API accepts **doubled-integer** inputs via a `_2j` suffix. The
`irrep_cg_2j(two_j1, two_m1, two_j2, two_m2, two_J, two_M)` signature
makes spin-½ unambiguous: the sign `⟨½, +½; ½, −½ | 1, 0⟩ = +1/√2` enters
as `irrep_cg_2j(1, 1, 1, −1, 2, 0) = +1/√2`, verified in
`tests/reference_data/cg_reference.h` against Sakurai Appendix A.

## 8. Recoupling: 6j, 9j, Racah W

### 8.1. Wigner 6j symbol

The 6j symbol `{j_1 j_2 j_3 / j_4 j_5 j_6}` encodes the coupling of three
angular momenta in two different orders. Its closed form (Varshalovich §9)
```
 {j_1 j_2 j_3 / j_4 j_5 j_6} =
 Δ(j_1 j_2 j_3) Δ(j_1 j_5 j_6) Δ(j_4 j_2 j_6) Δ(j_4 j_5 j_3)
 · Σ_k (−1)^k (k + 1)! / [ (k − j_1 − j_2 − j_3)! (k − j_1 − j_5 − j_6)! (k − j_4 − j_2 − j_6)! (k − j_4 − j_5 − j_3)! (j_1 + j_2 + j_4 + j_5 − k)! (j_2 + j_3 + j_5 + j_6 − k)! (j_1 + j_3 + j_4 + j_6 − k)! ] (8.1)
```
satisfies 24 distinct column-permutation symmetries (Edmonds §6). The
library computes (8.1) in log-gamma form and verifies four independent
symmetry relations plus tabulated hand values.

### 8.2. Wigner 9j symbol

The 9j symbol encodes the coupling of four angular momenta in two
different orders. It can be expressed as a single sum over 6j products
```
 {j_1 j_2 j_3 / j_4 j_5 j_6 / j_7 j_8 j_9} =
 Σ_k (2k+1) (−1)^{2k} · {j_1 j_4 j_7 / j_8 j_9 k} · {j_2 j_5 j_8 / j_4 k j_6} · {j_3 j_6 j_9 / k j_1 j_2}. (8.2)
```
(Varshalovich §10). Libirrep evaluates (8.2) directly.

### 8.3. Racah W coefficient

`W(j_1 j_2 J j_3 ; j_{12} j_{23}) = (−1)^{j_1 + j_2 + J + j_3} · {j_1 j_2 j_{12} / j_3 J j_{23}}`;
this is the Racah form of the 6j symbol preferred in atomic-physics
literature (Racah 1942; Fano & Racah 1959). Provided for convenience.

## 9. Time reversal

### 9.1. The antiunitary T operator

Time-reversal `T` is an **antiunitary** operator:
`⟨T ψ | T φ⟩ = ⟨φ | ψ⟩ = ⟨ψ | φ⟩*`, and for scalars `α ∈ ℂ`,
`T (α |ψ⟩) = α* · T |ψ⟩` (Sakurai §4.4; Wigner 1932). It cannot be
represented by a matrix alone; the library encodes the **linear part**
`U_T` such that `T = U_T · K` where `K` is elementwise complex
conjugation in the basis under discussion.

### 9.2. T on spin-j irreps

For integer `l`, `T` acts as `K` in the complex-SH basis (up to a
convention-dependent phase). In the real-SH basis every integer-l irrep
becomes `T = I` — the identity — since real components map to themselves
under conjugation. `irrep_time_reversal_integer(l, out)` writes the
`(2l+1)×(2l+1)` identity.

For half-integer `j`, `T = i σ_y · K` on spin-½ (Sakurai Eq. 4.4.26); on
higher half-integer irreps `T` is the tensor power of the spin-½ form,
and libirrep builds the explicit matrix via
`irrep_time_reversal_half_integer(two_j, out)`.

### 9.3. T² and Kramers degeneracy

The composition `T² = (U_T · K)² = U_T · U_T*` is diagonal by an overall
phase. Direct computation gives
```
 T² = (+1) · I on any integer-l subspace,
 T² = (−1) · I on any half-integer-j subspace (9.1)
```
(Sakurai Eq. 4.4.42; Haake 2010, §2). The half-integer case — **Kramers'
theorem** (Kramers 1930) — forces every energy eigenstate of a
time-reversal-invariant Hamiltonian with half-integer total spin to be
at least doubly degenerate, a fact consequential for magnetic systems
and the design of time-reversal-equivariant neural networks.

Two multiset-level query functions are available. The original
`irrep_time_reversal_square_sign(const irrep_multiset_t *)` takes the
integer-l multiset and therefore returns `+1` on every non-empty input
— integer-l systems cannot carry Kramers degeneracy by construction.
The `_2j` variant
`irrep_time_reversal_square_sign_2j(const irrep_multiset_2j_t *)`
(header `irrep/multiset_2j.h`) takes the doubled-integer multiset and
returns `−1` whenever any block has odd `two_j` (half-integer spin),
`+1` on integer-only content, and `0` on empty input, directly
matching (9.1). Half-integer block matrices themselves are built by
`irrep_time_reversal_half_integer(two_j, out)`.

## 10. Parity in tensor products

Under a rotation, two irreps `(l_1, p_1)` and `(l_2, p_2)` couple into the
direct sum
```
 (l_1, p_1) ⊗ (l_2, p_2) = ⊕_{l_c = |l_1 − l_2|}^{l_1 + l_2} (l_c, p_1 · p_2). (10.1)
```
Parity multiplies (Edmonds §5.3; Tinkham 1964 §5). This implies, for
instance, that `(1, −1) ⊗ (1, −1) = (0, +1) ⊕ (1, +1) ⊕ (2, +1)`: the
cross product of two polar vectors is an **axial** pseudovector, not a
polar one. Libirrep enforces (10.1) at path-enumeration time
(`irrep_parity_filter_paths`), so a caller specifying
`1x1o → 1x1o` with inputs `1x1o ⊗ 1x1o` will see the target silently
produce zero paths — the parity-forbidden path is dropped rather than
raising an error, consistent with the "pure-math functions return
mathematical defaults" error-handling convention.

### 10.1. The `i^{l_1 + l_2 − l_c}` phase factor

Real-basis tensor-product coefficients obtained by direct change of
basis from the complex CG pick up a factor `i^{l_1 + l_2 − l_c}` when
`l_1 + l_2 − l_c` is odd. Libirrep's `irrep_tp_build_uvw` applies this
phase internally so that the real-basis output is guaranteed real for
every valid path, including odd-l-sum paths like the cross-product
`(1, 1, 1)` (see `examples/torque_net_tp_paths.c` for a machine-precision
cartesian round-trip demonstrating this).

## 11. Point-group irreps

For a finite subgroup `G ⊂ O(3)`, the `(2l+1)`-dimensional SO(3) irrep
restricts to a **reducible** representation of `G`. The decomposition
into irreps of `G` (character labels `A₁, A₂, B₁, B₂, E, E₁, E₂, T₁, T₂, …`)
follows from the orthogonality relation
```
 m_μ = (1 / |G|) · Σ_{g ∈ G} χ_μ(g)* · χ_V(g), (11.1)
```
where `χ_μ(g)` is the character of irrep `μ` at element `g` and
`χ_V(g) = Tr(D_V(g))` is the character of the restricted SO(3) rep. For
a rotation `R(n̂, θ) ∈ SO(3)` the l-irrep character reduces to
```
 χ_l(θ) = sin((2l + 1) θ / 2) / sin(θ / 2), (11.2)
```
the **character of the irreducible spin-l representation** (Weyl 1928;
Tinkham §5.6). Improper elements pick up an additional `(+1)` or `(−1)`
depending on the irrep's parity (Bradley & Cracknell 1972 §1.7).

### 11.1. Projection operator

The projector onto the `μ`-th irreducible subspace is
```
 P_μ = (d_μ / |G|) · Σ_{g ∈ G} χ_μ(g)* · D(g), (11.3)
```
where `d_μ` is the dimension of `μ`. This satisfies `P_μ² = P_μ`,
`P_μ P_ν = 0` for `μ ≠ ν`, and `Σ_μ P_μ = I` (projector idempotence and
completeness; Hamermesh 1962 §3.7). Libirrep verifies all three relations
to `1e-10` in `tests/test_point_group.c`.

### 11.2. Supported groups (1.2)

| Group | Order | Irreps | Character-table source |
| ----- | ----: | ------ | ---------------------- |
| C₄ᵥ | 8 | A₁, A₂, B₁, B₂, E | Bradley & Cracknell 1972, Table 3 |
| D₆ | 12 | A₁, A₂, B₁, B₂, E₁, E₂ | Bradley & Cracknell 1972, Table 3 |
| C₃ᵥ | 6 | A₁, A₂, E | Bradley & Cracknell 1972, Table 3 |
| D₃ | 6 | A₁, A₂, E | Bradley & Cracknell 1972, Table 3 |

All tables verified against Altmann & Herzig (1994) as an independent
source before commit.

## 12. Numerical conventions

### 12.1. Precision defaults

Double precision (IEEE 754 binary64) throughout; `_f32` variants exist
for mixed-precision pipelines and currently wrap the double kernel with
a cast (a real single-precision SIMD path is scheduled for 1.3). Every
unit test asserts double-precision results to `1e-10` relative error or
better; the spherical-harmonic and CG paths lock to machine precision
(≲ 5 × 10⁻¹⁴) where direct closed forms exist (`tests/test_spherical_harmonics.c`,
`tests/test_clebsch_gordan.c`).

### 12.2. FP contraction

The ISO C11 `#pragma STDC FP_CONTRACT` state is **OFF** in
`src/spherical_harmonics.c` and `src/spherical_harmonics_neon.c` so that
the NEON batch kernel matches the scalar kernel bit-for-bit, regardless
of the compiler's choice to fuse `a*b − c*d` into `fmsub`. Elsewhere the
default `ON` applies; consumers whose own CFLAGS include
`-ffp-contract=fast` will see FMA-fused code in the rest of the library
and should be aware that the 1-ulp variance this introduces is within
every test tolerance.

### 12.3. Thresholds with derivations

| Threshold | Value | Derivation |
| ----------------------- | ------- | ------------------------------------------------------------------------------- |
| `IRREP_EULER_GIMBAL_EPS` | 10⁻¹² | Below this `|sin β|`, `sin β · (cos α ± 1)` is indistinguishable from 0 at double precision after the assembly roundoff accumulated by the three Euler multiplications. |
| `IRREP_ROT_EXP_SMALL` | 10⁻¹² | Rodrigues divides `(1 − cos θ)/θ²` and `sin θ / θ`; truncating to the linear term `I + [ω]_×` incurs O(θ²) error, below `10⁻²⁴` in that regime. |
| `IRREP_SH_POLE_EPS` | 10⁻¹⁴ | At the pole (`r_xy → 0`), `x/r_xy` divides by a near-zero quantity; below this threshold we fall back to `cp = 1, sp = 0` — the correct limit since `Y_{l, m≠0}` carries a `(sin θ)^{|m|}` prefactor that vanishes there. |
| Bessel-RBF small-r | `ar < 10⁻³` | The naive derivative `a cos(ar)/r − sin(ar)/r²` loses up to 14 digits to catastrophic cancellation at `ar ≈ 10⁻⁷`; a three-term Taylor `−C a³ r / 3 · (1 − (ar)²/10 + (ar)⁴/280)` is accurate to `10⁻¹⁴` relative at `|ar| ≤ 10⁻³`. |
| Karcher-Fréchet `tol_sq` | 10⁻²⁴ | Square of `10⁻¹²` rad — below the angular resolution of a unit quaternion in double precision. |

Each threshold is named with a macro or a block comment in the source so
that it is tractable to audit.

## 13. Bond exchange tensor symmetry

The full bilinear two-spin coupling on a bond `(i, j)` is

```
    H_ij = S_i · J_ij · S_j        (J_ij a 3×3 real matrix, 9 components)
```

The symmetric / antisymmetric decomposition is

```
    J_ij^{ab} = J^s_ij^{ab} + J^a_ij^{ab}
              = ½(J_ij^{ab} + J_ij^{ba})  +  ½(J_ij^{ab} - J_ij^{ba})
```

The antisymmetric part has three independent components and is
encoded as the **DMI vector** `D_ij` via

```
    J^a_{ij,ab} = ε_{abc} · D_ij^c            (Levi-Civita encoding)
    H^a_ij      = D_ij · (S_i × S_j)
```

The symmetric part has six independent components: one Heisenberg
(trace, transforms as `A₁` of every group) plus five traceless
anisotropic components transforming as the rank-2 symmetric
representation (= `Eg + T2g` under cubic).

### 13.1. Transformation under symmetry

Under a spatial symmetry operation `g` acting on cartesian space as
`r → det(g) · R(g) · r` (libirrep's `(R_proper, det)` convention,
where `R_proper` is the proper-rotation matrix and `det = ±1`
distinguishes proper from improper), spins transform as **axial
vectors**:

```
    S_i  →  R(g) · S_{g(i)}
```

with **no `det(g)` factor** — improper operations (mirrors, S_n,
inversion) act on the spin's vector value the same as their
proper-rotation counterpart. Inversion leaves the spin at its
original direction (since the position flip and the axial-vector
structure cancel).

For the cross product:

```
    (R u) × (R v) = det(R) · R · (u × v)
```

but since `R = R_proper` is proper (det = +1), we get
`(R u) × (R v) = R (u × v)` for spin transformations — the cross
product of two axial vectors transforms as `R` regardless of the
symmetry operation's parity. This is the cancellation that makes
DMI insensitive to `det(g)` in its matrix-element form.

### 13.2. DMI: Moriya's five rules

For invariance of `H = D_ij · (S_i × S_j)` under operation `g`:

- If `g` **preserves** the bond as an ordered pair (`g(i) = i`,
  `g(j) = j`), the constraint is `D_ij = R(g) D_ij`. `D_ij` must lie
  in the +1 eigenspace of `R(g)`.

- If `g` **reverses** the bond (`g(i) = j`, `g(j) = i`), then
  `D_g(i)g(j) = D_ji = -D_ij` (antisymmetric exchange), and the
  constraint becomes `D_ij = -R(g) D_ij`. `D_ij` must lie in the
  −1 eigenspace of `R(g)`, i.e., the +1 eigenspace of `-R(g)`.

The **bond's site-symmetry group** is the subgroup of the parent
crystallographic point group that maps `{r_a, r_b}` to itself as
an unordered set. The symmetry-allowed `D` subspace is the joint
+1 eigenspace of all operators

```
    M_g  =  +R(g)   if g preserves the bond
            -R(g)   if g reverses the bond
```

equivalently the range of the projector

```
    P  =  (1/|S|) · Σ_{g ∈ S} M_g
```

where `S` is the bond's site-symmetry group. **The dimension of
the allowed D-subspace is `Tr(P)`.**

Specialising this projector to the five Moriya cases recovers his
1960 rules:

| case | g | role | M_g | constraint | D-direction |
|------|---|------|------|------------|-------------|
| (a)  | inversion at midpoint | reversing, `R = I` | `−I` | `D = −D` | `D = 0` |
| (b)  | mirror ⊥ bond, midpoint on plane | reversing, `R = R(n_b, π)` | `−R(n_b, π)` | `D ⊥ n_b` | D in mirror plane |
| (c)  | mirror containing bond | preserving, `R = R(n_M, π)` | `+R(n_M, π)` | `D ∥ n_M` | D ⊥ that plane |
| (d)  | C₂ ⊥ bond | reversing, `R = R(n_⊥, π)` | `−R(n_⊥, π)` | `D ⊥ n_⊥` | D ⊥ axis |
| (e)  | C_n along bond, n ≥ 2 | preserving, `R = R(n_b, 2π/n)` | `+R(n_b, 2π/n)` | `D ∥ n_b` | D ∥ bond |

In each row, `n_b` is the bond direction and `n_M` is the mirror
plane normal. The analyzer in `irrep/dmi.h` constructs `P` exactly
this way and returns its +1 eigenspace as an orthonormal basis.

### 13.3. Symmetric exchange tensor

For `J^s_ij` the constraint under `g` is

```
    J^s_ij  =  R(g) · J^s_ij · R(g)^T
```

for **both** preserving and reversing operations: bond reversal
acts as `J^s_ji = (J^s_ij)^T = J^s_ij` (symmetric tensor is its
own transpose), so the constraint coincides with the preserving
case. Improper operations contribute the same constraint as their
proper-rotation counterparts (the `det(g)` factors cancel for a
rank-2 tensor of axial vectors). Consequently **the centrosymmetric
group `O_h` and its chiral subgroup `O` impose identical
constraints on the symmetric exchange of any bond** — the asymmetry
between the two groups appears only in the DMI sector.

The 6-dimensional space of symmetric 3×3 matrices is spanned by

```
    e_1 = diag(1, 0, 0)            e_4 = (E_xy + E_yx) / √2
    e_2 = diag(0, 1, 0)            e_5 = (E_xz + E_zx) / √2
    e_3 = diag(0, 0, 1)            e_6 = (E_yz + E_zy) / √2
```

orthonormal under the Frobenius inner product
`⟨A, B⟩ = trace(A^T B)`. The 6×6 representation matrix of the
operator `J → R J R^T` in this basis is

```
    M_g[β, α]  =  trace( e_β^T · R(g) · e_α · R(g)^T ).
```

Averaging over the bond's site-symmetry stabiliser gives the 6×6
projector `P_sym = (1/|S|) Σ_g M_g`. Its +1 eigenvectors, expressed
back in the `e_α` basis, are the symmetry-allowed J^s matrices.

### 13.4. Worked examples reproduced from group theory

The analyzer reproduces standard textbook results from
crystallographic input alone, with no material data:

**B20 chiral magnet (MnSi-type, P2_1 3, magnetic site sym = T or C₃)**.
For the Mn–Mn NN bond, the only relevant element of T preserving the
bond is the C₂' along the bond direction (in the parent O analysis
of the libirrep cubic group). By rule (e), `D ∥ bond` — the
**Bak-Jensen pattern** (Bak & Jensen 1980). The analyzer returns
`D · bond̂ = 1.000000` to six digits.

**Pyrochlore NN bond (Fd-3m, magnetic site sym = D₃d via inversion
at midpoint)**. The bond {(0,0,0), (¼,¼,0)} has inversion at
midpoint within the parent O_h, so by rule (a), `D = 0` for
NN DMI on bulk pyrochlores. The symmetric tensor J^s remains
3-dimensional, parametrised by the **Curnoe-Ross-Kao** components
(J_zz, J_±±, J_z±) that drive quantum-spin-ice phase diagrams in
Tb₂Ti₂O₇ / Yb₂Ti₂O₇ / Er₂Ti₂O₇.

**Kagome NN bond (D_6h with σ_h, layered metallic kagome magnets)**.
The σ_h forbids in-plane DMI (mirror containing bond → D ⊥ that
plane → D ∥ z). Stacking-broken bilayer kagome (Fe₃Sn₂ at room
temperature) breaks σ_h and recovers in-plane DMI, driving the
observed RT skyrmion-bubble texture.

These derivations are reproduced by `examples/dmi_pyrochlore_pattern.c`
and `examples/dmi_kagome_pattern.c` and verified by the per-rule
test sequence in `tests/test_dmi.c`.

### 13.5. Connection to crystallographic databases

The bond-exchange-tensor decomposition is the irreducible
group-theoretic step in a materials-design pipeline:

1. Crystallographic database (ICSD, COD, MAGNDATA) returns a
   candidate space group and Wyckoff positions for the magnetic
   sublattice.
2. libirrep's lattice + point-group infrastructure encodes the
   relevant subgroup.
3. The `dmi.h` analyzer returns the symmetry-allowed `(D, J^s)`
   basis vectors / matrices for each bond class.
4. DFT (VASP, Quantum Espresso, Wannier-projected hopping) provides
   the magnitude on each allowed component.
5. Micromagnetic codes (mumax, OOMMF) ingest the parameterised
   exchange tensor field and simulate skyrmion / domain-wall
   dynamics.
6. Experimental validation closes the loop.

Step 3 is the crystallographer's hand-derivation from International
Tables for Crystallography vol. A (Hahn 2005) plus Moriya 1960. The
analyzer automates it, which is the prerequisite for high-throughput
candidate-space-group **enumeration** rather than one-material-at-a-
time analysis.

## 14. Magnetic point groups (Shubnikov framework)

The point groups in §11 and the bond exchange tensor analysis in §13
treat only spatial (unitary) symmetry operations. Real magnetic crystals
have an additional **antiunitary** symmetry — time reversal `T` —
that combines with each spatial element `g` to give the full magnetic
point group structure. There are 122 magnetic point groups in 3D
(Heesch 1929, Shubnikov 1951) and 1651 magnetic space groups in
3D (Belov 1957). libirrep's analyzers handle only the spatial subgroup
as of 1.3.0-alpha; magnetic-point-group support is the natural next
extension.

### 14.1. Antiunitary operations

Time reversal `T` acts as

```
    T |s⟩  =  |s*⟩    (complex conjugate the wave-function amplitude)
    T S_i  =  −S_i    (flip the spin axial vector)
    T r_i  =  +r_i    (positions are real, unchanged)
```

so `T²` = `+1` for integer spin and `T²` = `−1` for half-integer spin
(Kramers — see §9.3). For the **spin-½ Heisenberg Hamiltonian**
`H = J Σ S_i · S_j`, `T` is a manifest symmetry (each `S` flips,
the dot product is invariant); the magnetic ground state may
spontaneously break it, but the Hamiltonian itself does not.

A **magnetic point-group element** is either `g` (proper or improper
spatial) or `T · g` (antiunitary). The composition rules:

```
    (g₁) · (g₂)         = g₁ g₂              (unitary × unitary = unitary)
    (g₁) · (T · g₂)     = T · (g₁ g₂)        (T commutes with spatial g)
    (T · g₁) · (g₂)     = T · (g₁ g₂)
    (T · g₁) · (T · g₂) = T² · (g₁ g₂) = ±(g₁ g₂)  (sign from Kramers)
```

For integer spin `T² = +I`, the magnetic point group has a structure
`G + T·G` where `G` is the spatial subgroup (sometimes the
"halving subgroup"). For half-integer spin `T² = −I`, the
"double magnetic group" doubles the order again.

### 14.2. The 4 types of magnetic point groups

For each ordinary point group `G`, there are up to 4 distinct
magnetic point groups:

1. **Type I** (`G`): no `T`-augmented elements. Equivalent to the
   ordinary point group. Describes purely-magnetic FM order
   that breaks `T`.

2. **Type II** (`G ⊕ T·G`, sometimes written `G 1'`): `T` is a
   group element on its own. Describes paramagnetic / non-magnetic
   crystals.

3. **Type III** (`G ⊕ T·H`): `T` combines with a halving subgroup
   `H ⊂ G` of index 2. The Shubnikov-grey type that drives the
   topological Hall effect in compensated AFMs (Mn₃Sn family).

4. **Type IV** (`G ⊕ T·t`): non-symmorphic — `T` combines with a
   half-translation, present only in magnetic *space* groups.
   Drives commensurate AFM ordering with a doubled magnetic unit cell.

Crystallographic databases (MAGNDATA at the Bilbao Crystallographic
Server) list materials by magnetic space group (BNS / OG notation).
The libirrep substrate at 1.3.0-alpha covers spatial subgroups only
— working out which magnetic point group augmentation a candidate
material has is left to the user; the predicted symmetry-allowed
exchange tensors then specialise (some couplings are killed by the
magnetic structure beyond what the spatial group predicts).

### 14.3. DMI under antiunitary symmetry

Time reversal flips the spin and complex-conjugates the
wave-function. The DMI Hamiltonian `H_DMI = D · (S_i × S_j)`
under `T`:

```
    T H_DMI T⁻¹  =  D · ((−S_i) × (−S_j))  =  +D · (S_i × S_j)  =  H_DMI
```

so DMI is `T`-symmetric (the two sign flips cancel). Pure time
reversal alone imposes no constraint on `D`.

For a combined operation `T · g`:

- If `g` preserves the bond: `T·g` does too. The constraint on `D`
  is `D = M_g D` for the spatial part; the `T` factor adds a sign
  flip from the antiunitarity, so the effective constraint is

  ```
      D  =  −M_g · D     (T·g preserving)
  ```

  the **opposite sign** from the unitary case.

- If `g` reverses the bond: `T·g` reverses the bond too. Sign flip
  from antiunitarity vs. sign flip from bond reversal cancel:

  ```
      D  =  +M_g · D     (T·g reversing)
  ```

For the symmetric exchange tensor, both signs cancel pairwise and
the constraint is identical to the unitary case (the analyzer's
output for `J^s` does not change when antiunitary elements are
added — consistent with the result that the same `J^s` subspace
emerges under O_h and O even though their unitary structures
differ in inversion).

### 14.4. Implementation pathway

Adding magnetic-point-group support to `dmi.h` is a small extension:

1. Augment the `irrep_dmi_sym_op_t` struct with an `antiunitary`
   flag (one extra bool per element).
2. In the analyzer's projector accumulator, flip the sign rule:

   ```
       M_g  =  +R   (preserving, unitary)     OR  +R · sign_T   (preserving, antiunitary)
              −R   (reversing, unitary)        OR  −R · sign_T   (reversing, antiunitary)
   ```
   where `sign_T = −1` for the `T` axial-vector flip.
3. Add a magnetic-point-group enum sweep (122 groups) populating
   the standard halving-subgroup augmentations.

The first two steps are ~30 lines of code; the third is data entry
from Bradley-Cracknell vol. 2. For the materials-search pipeline,
this would let the analyzer return DMI subspaces specialised to a
specific candidate magnetic ordering — not just "what spatial
symmetry allows" but "what magnetic structure permits given the
ordering vector and antiunitary symmetry of the candidate phase".

References for §14: Bradley-Cracknell vol. 2 (1972); Wills, Lavrov &
Ouladdiaf review *Phys. Reports* (2010); MAGNDATA / Bilbao
Crystallographic Server (Gallego, Perez-Mato et al., 2016).
